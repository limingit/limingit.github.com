Activity

Android 有四大组件：Activity、Services、Content Providers 和 Intents。

Activity（活动窗口）在应用程序中提供用户交互的一个窗口。每个 Activity 都绘制自身的 UI 控件。一个应用通常包含多个 Activity，其中必然有一个主 Activity，应用初次启动的时候进入到该 Activity。每个 Activity 都能启动其他的 Activity。新的 Activity 启动的时候，之前的 Activity 停止，但系统会将前者放入一个栈中等待用户唤起。

当一个 Activity 由于切换到新的 Activity 而停止的时候，系统会通过 Activity 的生命周期事件来发出相应的通知。

布局

布局文件主要用于规范和设计用户界面（UI），采用 XML 开发，实现与逻辑代码的分离。布局文件位于工程的 res/layout 目录。

打开布局文件，首先看到的是界面设计面板，点击下方的“activity_main.xml”，可以查看对应的源代码。

Android布局包含五种：相对布局（RelativeLayout）、线性布局（LinearLayout）、表格布局（TableLayout）、帧布局（FrameLayout）、网格布局（GridLayout）。新建工程的布局，默认为相对布局（RelativeLayout）。

Android 设备尺寸众多，分辨率也五花八门，因此不建议使用绝对布局。

布局的距离单位有：设备独立像素dp、像素px、放大像素sp。

  • dp，即 dip（device independent pixels，设备独立像素），在不同设备上显示效果不同，与设备硬件有关。一般为了支持 WVGA、HVGA 和 QVGA，推荐使用此单位，因为它不依赖像素。
  • px，即 pixels（像素），在不同的设备上显示效果相同。一般用 HVGA 代表 320*480 像素
  • sp，即 special pixels（放大像素），主要用于字体的显示

一般来说，表示长度、高度等用 dp，设置字体大小用 sp。

布局常用属性：

  • id，布局文件的唯一标识
  • layout_width，布局的宽度
  • layout_height，布局的高度

相对布局

相对布局包括相对容器布局和相对控件布局。

表格布局

表格布局以行和列的形式，将界面划分成多个单元格。表格的每一行为一个 TableRow，每当一个控件添加到 TableRow 中就生成一个单元格。表格中的每一行可以有 0 到多个单元格，一个单元格可以跨越多列。

表格布局针对列的设置提供了三个特有属性：stretchColumns（指定该列被拉伸）、shrinkColumns（指定该类被收缩）、collapseColumns（指定该列被隐藏），列号是一个数字，从 0 开始。

帧布局

帧布局是最简单的布局方式，但是使用的极少。所有控件会在左上角叠加显示，帧布局为每个加入的控件创建一个空白区域，非常类似于 Photoshop 的层。

网格布局和布局组件

网格布局是 Android 4.0 新增的布局，它实现了控件的交错显示，能够避免使用之前的布局嵌套，对设备性能的影响大大减少，更有利于自由布局的开发。

网格布局使用一组无限细的直线将绘图区域分隔成行、列和单元，并指定控件的显示区域和控件在该区域的显示方式。

新增的属性： * android:columnCount 列数 * android:rowCount 行数 * android:orientation 布局的显示方式，有 horizontal 和 vertical 两种，默认是 horizontal。 * android:layout_column 控件所在的列 * android:layout_row 控件所在的行 * android:layout_columnSpan 控件跨越的列数 * android:layout_rowSpan 控件跨越的行数 * android:gravity 控件的位置：left、right、top、bottom、center、center_horizontal（水平居中）、center_vertical（垂直居中）、fill（填充所有网格）、fill_vitical（填充 1 列）、fill_horizontal（填充 1 行）

基本控件

控件是界面的主要组成元素。Android 界面控件分为系统控件和定制控件两种。前者是 Android 系统提供给用户的已经封装的界面组件，包括应用程序开发过程中常用的功能控件。系统控件可以帮助用户进行快速开发，并能够使 Android 系统应用程序的界面保持一致。定制控件是用户独立开发的控件，或者通过继承并修改系统控件而产生的新控件。

Android 控件一般在 res/layout 目录下的布局文件中声明，每个空间都有自己的 id 及在布局中的宽高（layout_width 和 layout_height），此外每个控件还有自己特有的属性。
