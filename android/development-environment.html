开发环境

  • JDK + Eclipse + Android SDK + ADT 下载并安装 JDK 配置 Java 环境变量：新增 JAVA_HOME（指向 JDK 根）、修改 PATH（指向 JDK 根下面的 bin） 下载安装 Eclipse 下载 Android SDK，配置 SDK 环境变量——修改 PATH，增加 SDK/tools 安装 ADT，Eclipse 主菜单 Help –> Install –> New Software，打开 Install 对话框， AVD创建

真机调试

  • 开启手机 USB 调试模式（红米手机的设置在“关于手机”里，连续点击 Android 版本四次，然后回到上一级，关于手机上面就出现了开发者选项）
  • 安装手机驱动 手机与手机公司网站的对应列表：http://developer.android.com/tools/extras/oem-usb.html ，不过有很多不准了，比如小米的应该是：http://xiazai.mi.com/
  • 配置 Run As 里的 Run Configuration，在 Target 页签下点击 Manual，Apply 后点击 Run，选择设备

目录结构

  • src 存放 Java 程序源代码
  • gen 保存自动产生的 R.java 文件，Java 代码中的 R.. 都是从这个静态类获取的
  • assets 存放外部资源文件
  • bin 存放生成的目标文件，如 Java 的二进制文件、资源打包文件、Dalvik 虚拟机的可执行文件等，还有 apk 文件等
  • libs 第三方库
  • res 存放图片等各种资源文件，比如：layout 存放布局文件，values 下存放各种 XML 格式的资源文件（string.xml 字符串资源文件、colors.xml 颜色资源文件、dimens 尺寸资源文件），drawable-* 存放图片文件，ldpi 表示低分辨率，mdpi 表示中分辨率，hdpi 表示高分辨率，xhdpi 表示超高分辨率。
  • AndroidManifest.xml 应用清单文件，也是整个程序的系统控制文件。它用于控制 Android 应用的名称、图标、访问权限等整体属性，应用本身的 Activity、Service、ContentProvider、BroadcastRecevier 这 4 大组件也需要在这里配置。

自动生成的 R.java

根据 R.java 类的源代码，我们可以看出如下几点： * 通过注释看到，这个文件是 AAPT 工具扫描应用 res 目录中资源文件来自动生成的。请谨记，这个不能手动修改，否则可能导致一些未知的错误。 * 每个资源对应 R 类的一个内部类，比如布局都对应 layout 这个内部类，字符串资源对应 string 内部类，标识符资源对应于 id 内部类。 * 每个具体的资源向都对应于内部类的一个静态 Field。比如访问应用名称，可以通过 R.string.app_name 获得。

res 目录说明

res 目录是一个特殊的文件夹，里面存放了应用所用的全部资源，包括图片、字符串、颜色、尺寸、id、布局文件等。

Android 应用规定，将不同的资源放在不同的文件夹内，这样可以方便的让 AAPT 工具来扫描这些资源，并为它们生成对应的资源清单类：R.java。

上面的资源文件中定义了 3 个字符串常量，标签中间是常量的值，name 属性指定字符串常量的名称。一旦定义了这个资源文件，就可以在布局文件的 XML，或者 Java 代码中使用它了。

  • Java 代码中使用：R.string.app_name
  • XML 文件中使用：android:text=“@string/app_name”

标识符（id）是一个特例，没有一个专门的文件用于描述标识符，它一般都直接在布局文件的 XML 中定义：android:id=“@+id/ok”；调用的时候，Java 代码：this.findViewById(R.id.ok)，XML 中：android:layout_alignParentTop=”@id/ok”。

清单文件 AndroidManifest.xml

AndroidManifest.xml 清单文件是每个 Android 项目所必需的，它是整个应用的全局描述文件。

AndroidManifest.xml 通常包含如下信息： * 应用程序的包名，例如 com.begionios.firstapp。包名相同的应用重复安装，会被覆盖掉。 * 应用程序兼容的 Android 版本 * 应用程序需要使用的刺痛所需的权限声明 * 其他程序访问改程序所需的权限声明 * 应用程序包含的 4 大组件的描述信息，例如 Activity 等
