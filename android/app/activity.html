什么是 Activity

一款手机应用，用户能最直观感受到的，就是其用户界面。应用的各种功能也是用户通过在界面中的各种手势操作来完成。

通常一款应用，会有很多个不同的界面组成。这非常类似于“话剧”，话剧表演的舞台需要随着剧情的发展，切换到不同的场景。我们可以把每个手机操作界面，看做一个“场景”。其实在很多游戏里就是用的这个概念，比如 Cocos2D-X 中，就是用的 CCScene 类来表示场景。

Android 应用也通常由多个基本组件组成，其中有四大组件是最常见的：Activity、Service、BroadcastReceiver、ContentProvider。

每个 Activity 就是 Android 应用中的场景，它是负责与用户交互的组件。Activity 本身并没有可见的内容，任何一个场景都需要继承 Activity 类（android.app.Activity）来实现具体的场景。

有多少个用户界面，就需要有多少个 Activity，多个 Activity 组成一个 Activity 栈，当前活动的 Activity 位于栈顶。

Activity 需要在 OnCreate() 方法中，通过 this.setContentView() 方法给自己指定一个 View 组件，不然将显示为空白。这个 View 组件内，就可以放许多 UI 控件。

View 组件是所有 UI 控件、容器控件的基类，是用户可以实实在在看到的部分。View 组件可以放到容器组件中，或者直接通过 Activity 将它显示出来。

Intent 和 IntentFilter

严格地说，Intent 并不是 Android 应用的组件，但它对于 Android 应用的作用非常大——它是应用内不同组件之间的通信载体。当应用运行时，需要切换到不同的 Activity 的时候，就需要借助于 Intent 来实现。Intent可以启动 Activity、Service，也可以发送一条广播信息来出发系统中的 BroadcastReceiver。
